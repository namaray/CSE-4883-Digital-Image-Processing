1. Basic Relationships: Neighbors of a Pixel

Any image can be represented on a coordinate grid. Let a pixel p be located at coordinates (x, y).

(x-1, y+1) (x, y+1)
Top		 (x+1, y+1)
(x-1, y)
Left		 p (x, y) (x+1, y)
Right
(x-1, y-1) (x, y-1)
Bottom		 (x+1, y-1)

Categories of Neighbors:

  • 4-Neighbors ( N4(p) ): The horizontal and vertical neighbors.
    Coordinates: (x+1, y), (x-1, y), (x, y+1), (x, y-1).
  • Diagonal Neighbors ( ND(p) ): The four diagonal neighbors.
    Coordinates: (x+1, y+1), (x+1, y-1), (x-1, y+1), (x-1, y-1).
  • 8-Neighbors ( N8(p) ): The union of 4-neighbors and diagonal neighbors.
    Formula: N8(p) = N4(p) + ND(p)

2. Adjacency & Connectivity

⚠️ EXAM TRAP: Neighbor vs. Adjacent

Just because two pixels touch (are neighbors) does NOT mean they are adjacent!
Definition: Two pixels are adjacent ONLY IF they are neighbors AND their intensity values belong to a specific set of allowed values, denoted as V.

  • Binary Images: Pixels are adjacent if they are neighbors & have the same intensity values (e.g., V = {1}).
  • Grayscale Images: The set V might contain multiple values (e.g., V = {128, 129, 130...}).

Why do we need Adjacency Information? (Applications)

Adjacency defines how pixels/regions are connected. It is used heavily in these 5 areas:

  1. Object Segmentation: Determines which pixels belong to the same object (e.g., medical imaging - do scattered cancer cells form one tumor or multiple?).
  2. Edge Detection: Identifies boundaries between objects by checking pixel connectivity (e.g., lips vs. skin).
  3. Noise Removal: Isolates disconnected noisy pixels (like salt-and-pepper noise) from meaningful regions.
  4. Pathfinding (Robotics/AI): Navigating maps using 4-connected vs. 8-connected paths.
  5. Morphological Operations: Dilation/Erosion; adjacency defines how shapes grow or shrink.

3. Types of Adjacency (The Rules)

Let pixels p and q have values from the set V.

  • 4-Adjacency: p and q are 4-adjacent if q is in the set N4(p).
  • 8-Adjacency: p and q are 8-adjacent if q is in the set N8(p).

🔥 Mixed Adjacency (m-Adjacency) - GUARANTEED EXAM QUESTION

Two pixels p and q with values from V are m-adjacent if:

  • (i) q is in N4(p)
  • — OR —
  • (ii) q is in ND(p) AND the intersection N4(p) ∩ N4(q) has NO pixels whose values are from V.

Plain English Translation of Rule (ii): If p and q are diagonal to each other, look at the two pixels they share as common 4-neighbors. If both of those shared pixels have values that are NOT in set V, then p and q are m-adjacent. If even one shared neighbor is in set V, they are NOT m-adjacent. This prevents multiple path ambiguities.

4. Exam Walkthrough: Solving m-Adjacency

Given the set V = { 1 }. Let's analyze the following grid of pixels (labeled a through i for location reference):

0 a 1 b 1 c
0 d 1 e 0 f
0 g 0 h 1 i

Example 1: Are b and c m-adjacent?

  • Look at their positions: b is (top-mid), c is (top-right).
  • Are they 4-neighbors? Yes. c is immediately to the right of b.
  • Since condition (i) is met, they ARE m-adjacent.

Example 2: Are b and e m-adjacent?

  • Look at their positions: b is (top-mid), e is (center).
  • Are they 4-neighbors? Yes. b is directly above e.
  • Since condition (i) is met, they ARE m-adjacent.

Example 3: Are e and i m-adjacent?

  • Look at their positions: e is (center), i is (bottom-right).
  • Are they 4-neighbors? No, they are diagonal. We must check condition (ii).
  • Find their shared 4-neighbors: N4(e) ∩ N4(i). The pixels mutually touching both are f (mid-right) and h (bottom-mid).
  • Check the values of f and h. Pixel f = 0. Pixel h = 0.
  • Are either of these values in V={1}? No. Therefore, the intersection has no pixels from V.
  • Because the intersection is empty of V, they ARE m-adjacent.

Example 4: Are e and c m-adjacent? (The Tricky One)

  • Look at their positions: e is (center), c is (top-right).
  • Are they 4-neighbors? No, they are diagonal. We must check condition (ii).
  • Find their shared 4-neighbors: N4(e) ∩ N4(c). The pixels mutually touching both are b (top-mid) and f (mid-right).
  • Check the values of b and f. Pixel b = 1. Pixel f = 0.
  • Are either of these values in V={1}? YES. Pixel b has a value of 1.
  • Because the intersection contains a pixel from V, condition (ii) fails.
  • Therefore, they are NOT m-adjacent.
