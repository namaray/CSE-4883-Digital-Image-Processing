1. From Tables to Images: The Core Problem

Traditional ML used spreadsheet-like data (features in columns). For images, each pixel is a feature. A 256×256 RGB image has 196,608 features — too large for standard ML algorithms.

How Computers Interpret Images

  • Each image is composed of pixels, each with a value indicating brightness.
  • A grayscale image = a single 2D matrix of pixel values (0-255).
  • A color (RGB) image = 3 matrices stacked (Red channel, Green channel, Blue channel).

2. What is a Kernel?

A kernel (also called a filter) is a small matrix whose values are used to modify an image through convolution. The kernel slides across the image, and at each position, computes a weighted sum of the covered pixels.

  • In CNNs, the kernel values (weights) are learned during training — not manually set.
  • Different kernels detect different features: edges, textures, curves, etc.
  • Multiple kernels are used in parallel, each producing a separate feature map.

🎯 Kernel → Feature Map → Classification

The ML algorithm works by applying a series of learned kernels to the pixel matrix of an image, passing the results through activation functions, and finally producing a classification output. If the kernels and weights are properly trained, the output correctly identifies the content of the image.

3. CNN Architecture Components

Convolution Layer

  • Applies a kernel to the input by sliding it across the image and computing dot products at each position.
  • Spatial invariance: Anything detectable in one small region looks the same in another region — CNNs exploit this by sharing kernel weights.

Stride

The stride is the distance between successive pixels on which the kernel is centered.

  • Larger stride → reduces the output size (roughly from n to n/stride).
  • Controls how much spatial information is preserved vs. compressed.

Padding

Padding adds extra pixels (zeros or copies of border pixels) around the input image border to control the output size after convolution.

  • Zero padding: Adds rows/columns of 0s around the image.
  • Replicate padding: Copies outer pixel values into the border.

Multiple Input Channels

For a color (RGB) image, each kernel is 3D — it covers all 3 channels simultaneously. The kernel produces one 2D feature map as output per kernel applied.

Receptive Field

The receptive field is the region of input that influences a particular unit's output. Deeper layers have larger receptive fields, enabling detection of larger features.

Pooling

  • Average Pooling: Takes the average value within each window. When stride equals window size, it down-samples the image.
  • Max Pooling: Takes the maximum value within each window. Acts as a feature detector — preserves the strongest activation in a region.

Batch Normalization

Normalizes the activations in each mini-batch to have zero mean and unit variance:

ẑ = (z - μ) / √(σ² + ε)     y = γ * ẑ + β

Where μ = mean, σ = std deviation, ε = small constant, γ and β are learned parameters. Helps stabilize training and allows higher learning rates.

Tensors in CNN

Data in CNNs is represented as multi-dimensional arrays (tensors):

  • A minibatch of 64 images of 256×256 RGB = tensor of size 256 × 256 × 3 × 64
  • Applying 96 kernels of size 5×5×3 with stride 2: output = 128 × 128 × 96 × 64 (called a Feature Map)
  • Each channel in the feature map carries information from one learned feature.

4. CNN for Image Classification

The core insight: instead of manually providing pixel values as features to a linear model, we let the CNN automatically learn hierarchical features (edges → shapes → objects) from raw pixel data.

Training Process

  1. Feed image through CNN (Forward Pass) → get predicted class probabilities.
  2. Compare prediction to ground truth using a loss function (e.g., Cross-Entropy).
  3. Backpropagate the error through the network, adjusting weights using gradient descent.
  4. Repeat for all images across multiple epochs.

5. Object Detection Using CNN (YOLO Algorithm)

Object detection combines classification (what is it?) with regression (where is it?). The output includes class labels AND bounding box coordinates.

Bounding Boxes

A bounding box is an outline that highlights an object. Each bounding box has:

  • bx, by: Center coordinates of the bounding box
  • bw: Width of the bounding box
  • bh: Height of the bounding box
  • c: Class label (e.g., person, car, capsicum)

YOLO (You Only Look Once)

A popular real-time object detection algorithm based on CNN:

  • Divides the image into a grid of equal cells.
  • Each grid cell detects objects whose center falls within that cell.
  • For each cell, the network predicts: bounding box coordinates + confidence score + class probabilities.
  • Non-Maximum Suppression is applied to remove duplicate detections.

🎯 Pest Detection with Bounding Boxes (Final Exam Q1b)

  1. Apply a sliding window or region proposal method to generate candidate bounding boxes over leaf regions.
  2. For each bounding box region, apply CNN-based classification to detect pest damage (texture changes, bite marks).
  3. CNN kernels in early layers extract low-level features (edges, texture gradients), while deeper kernels capture complex patterns like bite mark shapes.
  4. Non-Maximum Suppression removes overlapping boxes, keeping the highest-confidence detection for each damaged area.
